## What NASDE does — in four steps

One `nasde run` command executes the whole chain.

<CardGrid>
	<Card title="1. Describe a task you understand" icon="pencil">
		An instruction, a repo snapshot, and the assessment criteria describing what a good solution looks like. The output can be anything the agent writes — code, a migration plan, an ADR, a SQL script, updated docs.
	</Card>
	<Card title="2. The agent solves it in a sandbox" icon="rocket">
		A safe, isolated environment — the agent can't touch your machine. Every run starts from the same clean state, so different configurations get a fair comparison. Powered by [Harbor](https://www.harborframework.com/), local Docker or cloud.
	</Card>
	<Card title="3. A reviewer agent assesses it" icon="approve-check">
		After the rough tests, a second coding agent (`claude` or `codex`) navigates the workspace and scores your chosen dimensions (e.g. *domain modeling*, *test quality*) on whatever scale you picked.
	</Card>
	<Card title="4. Results land in a dashboard" icon="bars">
		Browse scores, compare variants, and track how your agent setup evolves — optionally via [Opik](https://www.comet.com/site/products/opik/).
	</Card>
</CardGrid>

You're the one defining "what good looks like." NASDE just automates running the experiment and assessing it the same way every time.
